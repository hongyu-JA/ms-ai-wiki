## Test-Karten pro Produkt

### 🟢 GitHub Copilot App

```
Ziel:          Multi-Model-IDE + Multi-Agent-Coding-Squad selbst erleben
Voraussetzung: GitHub Copilot Pro/Business-Abo
Test:
  1. Copilot App von github.com/copilot/app downloaden
  2. Mit unserem Test-Repo (Ms-ai-wiki) verbinden
  3. Multi-Model-Switch testen: einen Bug mit GPT-5 fixen,
     dann mit Claude 4, Outputs vergleichen
  4. Multi-Agent-Workflow: Issue → Plan-Agent → Code-Agent → Test-Agent
  5. Falls Windows 12: Aion-Local-Model-Switch testen
Erfolgskriterien:
  - Modell-Wechsel funktioniert ohne Code-Änderung
  - Multi-Agent-Squad erledigt einen echten Task end-to-end
  - Latenz + Qualität pro Modell dokumentiert
Aufwand:       0.5 Tag
Output:        Erfahrungs-Notiz in GitHub Copilot App.md, status → ga
```

### 🟢/🟡 Work IQ (Beobachtung)

```
Ziel:          Verstehen, ob M365 Copilot bereits Work IQ nutzt
Voraussetzung: M365 Copilot Tenant (haben wir)
Test:
  1. In Copilot Chat fragen: "Was war mein letzter Austausch
     mit [Kollege] zum Thema X?"
  2. Beobachten: aggregiert Copilot über Mail+Teams+Files?
  3. Vergleichen mit reiner Graph-Suche
  4. Falls Custom-Agent möglich: Work-IQ-Endpoint in
     Copilot Studio testen
Erfolgskriterien:
  - Semantische Aggregation erkennbar (nicht nur Keyword-Match)
  - Berechtigungs-Trimming verifiziert (sieht nur Erlaubtes)
Aufwand:       0.5 Tag
Output:        Beobachtungs-Notiz, status → preview oder ga
```

### 🟡 MAI Models (Kern-POC — höchste Priorität)

```
Ziel:          MAI-Qualität vs GPT-4o vs Claude 4 für UNSERE Use-Cases
Voraussetzung: Azure-Subscription + Foundry-Zugang
Test:
  1. Foundry-Projekt anlegen, mai-large + mai-thinking-1 deployen
  2. Benchmark-Set aus 3 typischen Journai-Use-Cases:
     a) Schweizer Vertrags-Q&A (10 Fragen, Ground Truth)
     b) Code-Review (5 PRs, bekannte Bugs)
     c) Recherche-Synthese (3 Themen)
  3. Gleiche Prompts an mai-large, gpt-4o, claude-4
  4. Blind-Bewertung der Outputs (Qualität, Korrektheit, Stil)
  5. Latenz + Token-Kosten pro Modell messen
Erfolgskriterien:
  - Klares Qualitäts-Ranking pro Use-Case
  - Pricing-Vergleich mit echten Zahlen (nicht Keynote-Schätzung)
  - Empfehlungs-Update: wo MAI-First sinnvoll, wo nicht
Aufwand:       2 Tage
Output:        Benchmark-Tabelle in MAI Models.md, status → ga,
               DPIA-Template-Update für MAI-First-Kunden
```
